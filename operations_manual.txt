myHerd Ecosystem: Operations & Deployment Manual
==================================================

This manual serves as the definitive administrative guide for managing the cloud infrastructure, database syncing pipelines, and code deployment routines for the myHerdApp and myHerdFinance applications.

1. GitHub (Repository)
----------------------
* WHEN TO USE IT:
  Execute Git commands immediately after modifying, testing, or fixing your Python source scripts (such as 'app_finance.py', 'backup_finance.py', or 'requirements.txt') inside your VS Code workspace, right before you want those changes to go live on the internet.

* WHY WE USE IT:
  - Code Vault: Acts as a secure, cloud-based master backup of your entire codebase, protecting your development investment from local hardware failures.
  - Deployment Trigger: Acts as the deployment bridge between your local machine and the cloud. Pushing code to GitHub sends an automated webhook tracking signal to Render, telling it to pull the latest changes and rebuild your web application.

* QUICK-REFERENCE TERMINAL COMMANDS:
  Open your VS Code built-in terminal pane and execute the following commands sequentially:
  
  git add app_finance.py
  (Stages your modified files, packing them into the commit queue)
  
  git commit -m "Updated dashboard layout components"
  (Locks in your staged changes locally with a descriptive version log entry)
  
  git push origin main
  (Uploads the code directly to your GitHub cloud repository, launching the live build)
  Note: If your repository setup uses the older default branch layout, substitute 'main' with 'master' (git push origin master).


2. Render (render.com)
----------------------
* WHEN TO USE IT:
  Render runs automatically in the background. No manual terminal code commands are required for this platform. You only need to log into the web-based Render Dashboard using your browser when you want to monitor live deployment compilation logs, check server statuses, or verify that your web service is online.

* WHY WE USE IT:
  - Web Server Host: Streamlit and Flask applications require continuous runtime servers to be accessible over the internet. Render acts as your cloud server host, transforming your backend Python scripts into a live, public, mobile-responsive URL dashboard.
  - Continuous Integration: It automatically tracks your master GitHub branch. The moment a push is registered, Render pulls down the source, spins up a fresh virtual container environment, and updates your public web application endpoint.

* WEB INTERFACE ACCESS:
  Monitor production build lines and environment states directly via your browser dashboard:
  URL: https://dashboard.render.com
  Live URL: https://myherdfinance.onrender.com


3. Supabase (supabase.co)
-------------------------
* WHEN TO USE IT:
  Your automation scripts execute handshakes with Supabase every evening at the close of your daily farming operations. You only need to access the Supabase browser management interface manually when you want to audit the raw contents of your cloud storage storage container or inspect bucket privileges.

* WHY WE USE IT:
  - Permanent Cloud Vault: Render's containerized server instances are ephemeral and reset during routine updates, erasing all local disk changes. Supabase provides a permanent, secure, relational cloud-linked data bucket ('finance-vault') to house your master database file ('live_farm_finance.db').
  - Row-Level Security (RLS): Provides institutional-grade safety filters. While the live Render web app reads from this bucket to display live metric boards, any unauthorized database write requests are flatly rejected unless verified by your master 'service_role' cryptographic key header.

* QUICK-REFERENCE EXECUTION COMMAND:
  To run your automated ledger data sync bridge script, run the following command in your local terminal:
  
  python backup_finance.py
  (Executes the connection script that safely uploads your local database file to the Supabase storage bucket)


4. The pip Command (Python Package Installer)
---------------------------------------------
* WHEN TO USE IT:
  Execute this command within your local terminal environment exclusively when you need to install, upgrade, or remove an external Python third-party library or dependency that is not part of Python's default standard environment.

* WHY WE USE IT:
  - Package Delivery: Acts as Python's official automated software installation courier, fetching verified pre-built layout tools, data structures, and networking handlers from the global Python Package Index (PyPI), eliminating the need to write complex network stacks or user interfaces from scratch.

* QUICK-REFERENCE TERMINAL COMMANDS:
  To install development packages or update cloud build instructions, run the following commands:
  
  pip install streamlit pandas requests
  (Downloads and configures the core application framework libraries on your machine)
  
  pip freeze > requirements.txt
  (Crucial Step: Automatically writes the comprehensive inventory of your local dependencies and exact versions into 'requirements.txt'. This file must be pushed to GitHub so Render knows exactly what environments to mirror during cloud compilation)
